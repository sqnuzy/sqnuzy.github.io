字符串哈希与 vector 建图入门笔记

发表于 2026-04-16 分类于 ACM
从常见建图方式、DFS BFS 模板、lambda 写法到双哈希模板，整理一份更适合入门训练的笔记。

这篇笔记整理自 字符串哈希、vector建图.md，把原来比较跳跃的记录，重排成“适用场景 -> 模板 -> 易错点”的形式。

一、什么时候用 vector 建图

在入门和大部分普通比赛场景里，vector 邻接表已经够用了：

  • 写法直观
  • 维护方便
  • 支持多测时逐点清空
  • 修改边结构相对容易

常见几种建图方式可以简单理解成：

1. 邻接表

最常用，综合最均衡。多数题直接用它。

2. 邻接矩阵

适合点数很小、需要快速判断两点是否有边的场景。点数一大，空间会炸。

3. 链式前向星

更偏竞赛模板和卡常场景。一次建图、基本不改结构时很好用，但对新人来说可读性和维护成本都更高。

五、字符串哈希的核心理解

字符串哈希的本质，是把字符串映射成一个数值，从而快速比较两个子串是否相同。

最常见的用法：

  • 预处理前缀哈希
  • O(1) 求任意子串哈希
  • 配合二分、枚举、前后缀判断使用

为了尽量降低冲突风险，训练时建议直接双哈希。

七、几个容易踩的坑

  • 多测时，全局 vector 不能只 clear() 最外层，通常要按点清空。
  • 哈希数组下标从 1 开始时，记得给字符串前面补空格。
  • 只会背模板还不够，要知道自己比较的是“整串”还是“子串”。
  • 哈希相等通常表示“极大概率相等”，不是严格数学证明。

八、学习建议

如果你刚学到图论和哈希，不要急着把所有板子一次背完。更推荐的节奏是：

  1. 先能独立写出 vector 建图和 DFS BFS。
  2. 再把 lambda 递归写顺手。
  3. 然后做几道字符串哈希模板题。
  4. 最后再去总结哪些题是“哈希 + 二分”“哈希 + 前缀后缀”。

把这些基本功练熟之后，很多题会明显顺手很多。
