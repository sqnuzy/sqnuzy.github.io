25级第一次线上培训整理

发表于 2026-04-16 分类于 竞赛入门
把第一次线上培训中的零散要点按训练规范、基础知识和常用平台重新整理成一篇可长期维护的入门说明。

这篇文章整理自 25级第一次线上培训 10.26 的原始记录，保留重点信息，并补齐了更适合新生阅读的结构。

一、先说最重要的事

1. 禁止比赛作弊

这是训练和比赛中的底线问题，不管是平时周赛、线上赛还是正式比赛，都不要用不符合规则的方式获取答案或提交代码。

学算法竞赛最有价值的部分，不只是做出一道题，而是逐渐建立自己的思维方式、代码能力和复盘能力。作弊会直接破坏这件事。

2. 要有持续输出

培训里提到的“每周三次题解”，本质上不是形式要求，而是帮助自己完成复盘。写题解的过程会逼着你回答几个问题：

  • 这题到底卡在了哪里。
  • 我最后是怎么想到正解的。
  • 以后再遇到同类题，应该从什么方向入手。

如果只做题不总结，进步往往会很慢。

二、入门阶段应该补什么

1. C++ 基础要继续补

原始记录里特别提到了两件事：

  • 字符串，尤其是“带空格输入”和“多组字符串输入”。
  • 从 C 过渡到 C++。

这两个点非常关键。很多新生不是不会算法，而是会在输入输出、字符串处理、容器使用这些基础实现上频繁掉链子。

建议优先把这些内容补扎实：

  • string
  • vector
  • pair
  • sort
  • queue / stack
  • map / set
  • getline
  • 引用、结构体、函数封装

2. 对时间复杂度要有感觉

培训里提到一个很常用的经验值：

  • 一般情况下，1s 大约能跑 1e7 ~ 1e8 次基础操作。

这不是绝对标准，但足够作为入门判断依据。做题时至少要先能粗略区分：

  • O(n) 通常可行
  • O(n log n) 经常可行
  • O(n^2) 要看数据范围
  • O(2^n)、O(n!) 往往只适合很小的数据

三、日常训练怎么安排

1. 题解要认真写

题解可以用 Typora，也可以用其他 Markdown 编辑器。重点不是工具，而是内容有没有真正总结清楚。

比较推荐的题解结构：

  1. 题目链接
  2. 题目类型 / 知识点
  3. 自己第一次为什么没做出来
  4. 正确思路是什么
  5. 这题给自己的提醒

2. 推荐优先参加的比赛

原始资料里提到的几个比赛平台都很适合新人：

  1. Codeforces Div.2/3/4
  2. AtCoder Beginner Contest
  3. 牛客周赛 / 月赛

如果你刚入门，重点不是“打了多少高难比赛”，而是固定节奏参加、赛后补题、逐步提高通过题数。

四、常用网站整理

刷题与比赛平台

  • 牛客
  • AtCoder
  • Codeforces
  • 洛谷
  • Virtual Judge

刷题统计工具

  • CFTracker
  • AtCoder Problems
  • NowCoder Problems Tracker

比赛榜单

  • XCPC Board
  • RankLand

五、给 25 级同学的建议

1. 先稳定，再追难度

刚开始时，不要一上来只盯着难题或者“神仙题解”。更重要的是先把这些东西稳定下来：

  • 输入输出不出错
  • 常见 STL 会用
  • 搜索、前缀和、二分、简单图论能独立写
  • 赛后愿意复盘

2. 比“会不会”更重要的是“有没有积累”

很多知识点第一次看懂，不代表下次能写出来。真正有效的积累方式通常只有三件事：

  • 刷题
  • 补题
  • 写复盘

3. 环境问题要尽早解决

如果浏览器插件、VJudge、Typora 或本地编译环境经常出问题，训练节奏会被反复打断。建议尽快看完后面的工具整理文章，把环境先配顺手。

训练一旦形成稳定节奏，进步就会开始变快。
