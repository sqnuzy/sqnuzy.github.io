错题分析怎么写

发表于 2026-04-16 分类于 竞赛入门
基于训练记录整理一套更通用的错题分析模板，帮助把“做错”真正转化成可复用的经验。

原目录里的 zy错题分析.md 保留了很多真实训练记录。相比把整份原稿直接公开，我更想把它提炼成一套能长期复用的错题分析方法。

一、为什么要写错题分析

很多人做题时最大的问题不是“不会学”，而是“不会复盘”。

如果一道题只是看完题解、点点头、然后关掉页面，那下次遇到相似问题时，大概率还会卡在同一个地方。

错题分析的价值就在这里：

  • 逼自己说清楚卡点
  • 区分是知识漏洞还是思维漏洞
  • 给以后留下可检索的提醒

二、一份好用的模板

建议每次记录时至少写这几项：

1. 基本信息

  • 时间
  • 题目链接
  • 比赛 / 题号
  • 题目知识点

2. 当时为什么没做出来

这一段最重要，不要只写“不会”。

尽量写具体：

  • 没想到转化
  • 看错题意
  • 边界漏了
  • 模板不熟
  • 复杂度判断错
  • 想法对了但实现崩了

3. 正确思路是什么

不是照抄题解，而是用自己的话写：

  • 题目的关键转化是什么
  • 为什么这个做法成立
  • 哪一步是突破口

4. 以后再遇到同类题的提醒

比如：

  • 看到树上最远点，先想树的直径
  • 看到删连续段后前后字符数相等，先试前缀差值
  • 看到博弈且步数小，先想记忆化搜索

三、常见失误类型

从训练记录里，最常见的错因大致可以分成下面几类。

1. 没想到关键转化

表现是你已经会很多基础知识，但题目一变形就断思路。

解决方法通常不是再背模板，而是多做“同一类思想”的题，训练识别能力。

2. 知识点知道，但不够熟

比如知道树的直径、区间哈希、记忆化搜索这些概念，但一到现场就写不出来。

这种情况说明你缺的不是“知道”，而是“写过、错过、改过”。

3. 复杂度判断失误

有些题不是完全不会，而是一开始就把方向选错了，结果在错误复杂度上越写越深。

做题前先估复杂度，能帮你少走很多弯路。

4. 代码实现细节不稳

包括但不限于：

  • 数组没清空
  • 下标越界
  • long long 漏开
  • 多测没初始化
  • 交互题输出没刷新

这类问题不一定体现在题解里，但会直接影响通过率。

五、怎么把错题本真正用起来

错题分析不是写完就结束了，最好这样用：

  1. 每周回看一次本周错题。
  2. 把重复出现的错误单独归类。
  3. 对“总是不会”的知识点，找 3 到 5 道同类题专项补。
  4. 一个月后再回头看，看看自己是不是还会在同一点摔倒。

当你开始能准确描述“自己为什么错”时，进步通常就会快很多。
