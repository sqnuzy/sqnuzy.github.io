传递闭包

适用场景

求有向图中任意两点是否可达。

复杂度

  • Floyd：\(O(n^3)\)
  • bitset 优化：\(O(\frac{n^3}{w})\)，竞赛里通常可看成 \(O(n^3 / 32)\) 或 \(O(n^3 / 64)\)

注意点

  • 输入是邻接矩阵时最好直接做。
  • bitset 写法里，外层枚举中转点，内层枚举行点，且判断 f[i][j] 后做按位或。

树的直径

核心结论

  • 树上任意两点距离的最大值称为树的直径。
  • 从任意点出发，走到的最远点一定是某条直径的一个端点。
  • 求出直径端点 \(U, V\) 后，任意点 \(x\) 的最远距离为：

\[ \max(dist(x, U), dist(x, V)) \]

1. 两次 DFS / BFS

适用场景

边权非负，要求直径长度，或者还想顺手找直径端点。

复杂度

\(O(n)\)

注意点

  • 如果边权可能为负，不要用这个结论直接做。
  • 第一次找一个端点，第二次从该端点再跑一次找另一端点。

2. 树形 DP 求直径

适用场景

  • 边权可能为负
  • 需要嵌入树形 DP
  • 只关心直径长度

复杂度

\(O(n)\)

思路

设 d1, d2 表示当前点向下走的最长链和次长链，则经过当前点的最优答案为 d1 + d2。

3. 树的中心

树的中心一定在直径中点处。

  • 直径长度为偶数：中心有 \(1\) 个
  • 直径长度为奇数：中心有 \(2\) 个

4. 两棵树合并后的最小直径

若两棵树直径分别为 \(D_A, D_B\)，把两棵树的中心连起来最优。新直径为： \[ D_{new} = \max\left(D_A, D_B, \left\lceil \frac{D_A}2 \right\rceil + \left\lceil \frac{D_B}2 \right\rceil + 1\right) \]

最近公共祖先 LCA

1. 倍增

适用场景

  • 在线回答多次 LCA 询问
  • 也常配合树上距离、跳祖先、第 \(k\) 级祖先使用

复杂度

  • 预处理：\(O(n \log n)\)
  • 单次询问：\(O(\log n)\)

注意点

  • to[u][i] 表示 u 的 \(2^i\) 级祖先。
  • 先把深度拉平，再一起往上跳。
  • LOG 要根据数据范围开够。

2. Tarjan 离线 LCA

适用场景

所有询问提前给出，整题只离线处理一次。

复杂度

\(O(n + m)\)

注意点

  • 本质是 DFS + 并查集。
  • 只适合离线，不适合边读边答。

最小生成树

Boruvka 算法

适用场景

  • 每轮都能较快找到每个连通块向外的最小边
  • 适合一些特殊图、隐式图、补图、完全图变形

复杂度特征

每轮连通块数量至少大幅减少，因此轮数最多 \(O(\log n)\)。总复杂度取决于“每轮如何找每个块的最小出边”。

核心思想

开始时每个点单独成块。
每轮对每个连通块找一条最小出边，把这些边全部加入，然后合并连通块，直到只剩一个连通块。

树形换根 DP

适用场景

求每个点作为根时的答案，或者求“每个点到某类点的最远 / 最近距离”等信息。

这份模板解决的问题

有若干个被标记点，定义 \(f_i\) 为点 \(i\) 到所有被标记点距离的最大值，要求所有 \(f_i\) 的最小值。

核心思路

  • down[u]：从 u 往子树内走，能到达标记点的最远距离
  • up[u]：从 u 往父亲方向及兄弟子树方向走，能到达标记点的最远距离

最终每个点答案为： \[ \max(down[i], up[i]) \]

复杂度

\(O(n)\)

缩点

适用场景

有向图存在环，而题目要求“路径最优值”“最长路”“DP”等，通常先把强连通分量缩成 DAG 再做。

模板流程

  1. Tarjan 求 SCC
  2. 统计每个 SCC 的点权和
  3. 建新图 DAG
  4. 在 DAG 上拓扑 DP

复杂度

\(O(n + m)\)

注意点

  • Tarjan 中回边更新写成 low[u] = min(low[u], dfn[v])
  • 缩点后如果有重边，一般不影响 DAG DP，想去重可额外处理
  • “点权最大路径和”这类题，初值通常设为 SCC 自身权值
