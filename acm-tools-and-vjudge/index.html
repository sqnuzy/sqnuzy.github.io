工具环境与 VJudge 提交问题排查

发表于 2026-04-16 分类于 工具
整理常用写题工具、浏览器脚本环境以及 VJudge 提交异常时的常见处理步骤。

这篇文章把原目录中的 ScriptCat 插件教程、typora修改字体、VJ提交不成功解决办法 合并成了一份统一的工具说明。

二、ScriptCat 插件安装思路

原始资料里给的是截图版说明，这里改成文字步骤：

  1. 打开浏览器扩展管理页面。
  2. 打开开发者模式。
  3. 搜索并安装 ScriptCat。
  4. 如果社团内部提供了脚本压缩包，先解压，再在 ScriptCat 中选择导入。
  5. 导入完成后，确认脚本状态已经启用。

如果你后续需要用脚本辅助登录、同步或处理某些 OJ 页面，记得优先确认：

  • 插件是否已经启用
  • 脚本是否已经加载
  • 浏览器是否拦截了对应权限

三、VJudge 提交不成功时怎么排查

原始笔记的核心意思可以整理成下面几个步骤。

1. 先处理脚本文件

  • 如果拿到的是压缩包，先解压。
  • 确认里面是脚本或授权相关文件，而不是直接把压缩包原样导入。

2. 在对应 OJ 页面完成授权

原始资料里提到需要到对应站点页面完成设置，再回到 VJudge 使用。实际操作时可以按下面顺序检查：

  1. 登录对应 OJ 账号。
  2. 打开脚本或授权相关页面。
  3. 导入脚本 / 打开扩展。
  4. 根据页面提示完成一次授权或上报。

3. 如果页面里有 report 或类似按钮

原资料特别提到了 report 按钮。可以理解为：当脚本或授权状态已经变化后，需要主动刷新一次识别状态。

如果你看到页面已经出现授权成功、钥匙图标或者已连接的提示，通常就可以回到 VJudge 再次提交。

4. 还不行时重点检查这些地方

  • OJ 账号是不是登录错了
  • 浏览器扩展是不是被禁用了
  • 脚本是不是没有真正启用
  • 导入的是不是错误文件
  • 页面改版后脚本是否已经失效

四、训练环境的几个小建议

  • 比赛和训练时尽量使用固定的浏览器和固定的插件环境。
  • 输入法建议切回英文，避免全角符号影响代码。
  • 本地编译参数尽量固定，例如 -std=c++17 或比赛要求的指定版本。
  • 题解图片尽量放在可长期访问的位置，不要继续使用本机临时路径。

工具环境一旦稳定，平时写题和比赛时会省掉很多无意义的折腾时间。
