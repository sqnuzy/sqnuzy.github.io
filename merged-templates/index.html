一、动态规划

1. 数位 DP

1.1 数字计数

适用场景

求区间 \([a,b]\) 中每个数字 \(0 \sim 9\) 出现的总次数。

常见转化：

  • 先求 \([1,x]\) 中每个数码出现次数；
  • 再用 \(calc(b) - calc(a-1)\) 得到答案。

核心思路

设：

  • \(ten[i] = 10^i\)
  • \(dp[i]\) 表示在所有 \(i\) 位数的低 \(i\) 位贡献中，某个数字在每一位出现的总次数贡献

预处理：

\[ dp[i] = i \cdot 10^{i-1} \]

从高位到低位枚举当前位，统计：

  • 当前位左边固定时，右边自由变化带来的整段贡献；
  • 当前位取比原数当前位更小的值时带来的贡献；
  • 当前位恰好等于原数当前位时，右侧剩余部分形成的贡献；
  • 额外减去前导零贡献。

复杂度

  • 时间复杂度：\(O(\log_{10} n)\)
  • 空间复杂度：\(O(\log_{10} n)\)

注意点

  • 区间统计统一写成：calc(b) - calc(a - 1)。
  • 统计 0 时一定要处理前导零，模板里通过 cnt[0] -= ten[i - 1] 消掉非法贡献。
  • 当 a = 1 时，a - 1 = 0，要保证分解数字时不会出错。
  • 数据范围到 \(10^{12}\) 时，使用 long long。

模板

2.线性 DP

2.1 最长上升子序列 I

状态定义： f[i] 表示 以 a[i] 结尾 的最长严格上升子序列长度。

状态转移： 枚举前面的 j，如果 a[j] < a[i]，那么可以从 j 转移到 i： \[ f[i] = \max(f[i], f[j] + 1) \] 时间复杂度： O(n^2)

2.2 最长上升子序列 II

定义： q[len] 表示长度为 len + 1 的严格上升子序列的最小结尾值。

做法： 对于每个 a[i]：

  • 在 q 中找到第一个 >= a[i] 的位置
  • 用 a[i] 去替换它
  • 如果没找到，说明 a[i] 比当前所有结尾都大，可以直接接到后面

这样最后 q 的长度就是 LIS 长度。

二、数据结构

1.前缀与差分类

1.1 前缀置换

用途：维护前缀交换操作形成的排列，查询区间操作后的最终排列。
复杂度：预处理 \(O(10n)\)，单次查询 \(O(10\log 10)\)。

注意：这里只适合值域固定很小（这里是 10 个球）的情况。

1.2 前缀矩阵积

用途：2×2 转移矩阵区间连乘。
复杂度：预处理 \(O(n)\)，单次查询 \(O(1)\) 次矩阵运算。
注意：这里用到了矩阵求逆来实现区间矩阵积： \[ M_{[l,r]} = pre_{l-1}^{-1} \cdot pre_r \]

1.3 前缀和 + 等差数列

用途：区间贡献是 \(1,2,3,\dots\) 形式时，用多次前缀和实现。
复杂度：\(O(n)\)。

1.4 前缀和 + 平方数列

用途：区间后缀加 \(1,4,9,\dots\) 这类可由多次前缀和生成的序列。
技巧：

  • 常数列：1 次前缀和
  • 等差列：2 次前缀和
  • 平方列：3 次前缀和

1.5 前缀和 + 多项式

用途：静态批量区间加多项式，最后统一查询区间和。 核心：高阶差分 + 小次数多项式。 注意：这份代码是竞赛向写法，适合 \(k \le 5\)。

1.6 高阶前缀和 / 差分（NTT）

用途：对数组做 \(k\) 次前缀和 / 差分，\(k\) 可正可负且很大。 复杂度：\(O(n\log n)\)。 核心：组合数卷积。

1.7 SOS DP（子集和变换）

用途：查询某集合所有子集 / 超集的值和。 复杂度：预处理 \(O(n2^n)\)，查询 \(O(n)\) 或 \(O(1)\) 读答案。

2.区间数据结构

2.1 线段树（区间乘、区间加、区间和、平方和）

用途：同时维护

  • 区间和
  • 区间平方和
  • 区间乘法修改
  • 区间加法修改

复杂度：单次修改 / 查询 \(O(\log n)\)。 注意：懒标记是仿射变换 \(x \mapsto ax+b\)。

2.2 树状数组 1：单点加 / 区间和

复杂度：单次修改 / 查询 \(O(\log n)\)。

2.3 树状数组 2：区间加 / 单点查

技巧：维护差分数组的前缀和。 复杂度：单次 \(O(\log n)\)。

2.4 树状数组处理二维偏序

用途：把二维条件转成排序 + 一维树状数组。 套路：

  • 对第一维“排序”
  • 对第二维“离散化”
  • 用“树状数组”维护第二维

注意：

严格 > ：同 u 时，v 降序 非严格 ≥ ：同 u 时，v 升序

3.倍增与 RMQ

3.1 ST 表

用途：静态 RMQ。 复杂度：预处理 \(O(n\log n)\)，查询 \(O(1)\)。 注意：适合静态问题，不支持修改。

3.2 倍增表

用途：每点恰有一条出边时，求走 \(k\) 步后的信息。 可维护：

  • 第 \(2^j\) 步到哪
  • 权值和
  • 权值最小值

4.其他常用结构

4.1 三维前缀和

复杂度：预处理 \(O(n^3)\)，单次查询 \(O(1)\)。

4.2 字符串哈希（双模）

用途：判断两个子串是否相等。 复杂度：预处理 \(O(n)\)，查询 \(O(1)\)。 注意：双模更稳。

4.3 最小表示法

用途：求循环串中字典序最小的表示。 复杂度：\(O(n)\)。

4.4 rope

用途：大字符串 / 大序列的拼接、插入、删除。 复杂度：一般视作平衡树，单次操作近似 \(O(\log n)\)。 注意：GCC 扩展。

4.5 ordered_set

用途：支持求排名、求第 \(k\) 小。 注意：不能直接存重复值，若要存重复值可存 pair<值, 编号>。

4.6 普通可换根 Trie（统计前缀数量版）

用途：

  • 插入字符串
  • 查询某前缀出现次数
  • 整体加同一前缀字符（换根）

三、图论

1. 树的重心

定义

删除某个点后，剩余各个连通块的点数最大值最小，则这个点就是树的重心。

思路

设 sum 表示当前点 u 的子树大小。

删除 u 后，会形成若干连通块：

  • 每棵子树各自形成一个连通块，大小为对应子树大小
  • 除了 u 子树以外的其余部分也形成一个连通块，大小为 n - sum

取这些块大小的最大值，记为 mx。 所有点里 mx 最小的那个点就是重心。

模板

2. Bellman-Ford

有边数限制的最短路

适用场景

  • 图中可能有负边
  • 限制最多经过 k 条边
  • 求最短路

核心思想

进行 k 轮松弛。 每一轮表示允许再多走一条边。

这里一定要开 backup[] 备份上一轮的结果。 否则一轮中更新出来的新值又被继续拿去转移，就相当于一轮走了多条边。

模板

注意点

  • 一定要用 backup
  • 判断无解常用 res > INF / 2
  • 即使图里有负环，只要限制边数，这题仍然能做

3. SPFA

3.1 SPFA 求最短路

适用场景

  • 图中有负边
  • 没有负环
  • 求单源最短路

核心思想

SPFA 是 Bellman-Ford 的队列优化。
只有当前被更新过的点，才继续拿去松弛它连出去的边。

模板

注意点

  • st[v] 表示点 v 是否在队列中
  • 它不表示这个点已经确定最短路
  • SPFA 最坏复杂度仍然是 O(nm)

3.2 SPFA 判断负环

核心思想

如果某个点在最短路更新过程中，经过的边数 cnt[v] >= n，说明图中存在负环。

因为一个不重复经过点的简单路径最多只会经过 n - 1 条边。
如果还能继续更新，就说明绕了负环。

模板

注意点

  • 要把所有点一开始都入队
  • 因为负环不一定从 1 号点可达
  • 这是在判断整张图是否有负环

7. 树的直径

7.1 核心结论

  • 树上任意两点距离的最大值称为树的直径。
  • 从任意点出发，走到的最远点一定是某条直径的一个端点。
  • 求出直径端点 \(U, V\) 后，任意点 \(x\) 的最远距离为：

\[ \max(dist(x, U), dist(x, V)) \]

7.2 两次 DFS / BFS

适用场景

边权非负，要求直径长度，或者还想顺手找直径端点。

复杂度

\(O(n)\)

注意点

  • 如果边权可能为负，不要用这个结论直接做。
  • 第一次找一个端点，第二次从该端点再跑一次找另一端点。

7.3 树形 DP 求直径

适用场景

  • 边权可能为负
  • 需要嵌入树形 DP
  • 只关心直径长度

复杂度

\(O(n)\)

思路

设 d1, d2 表示当前点向下走的最长链和次长链，则经过当前点的最优答案为 d1 + d2。

7.4 树的中心

树的中心一定在直径中点处。

  • 直径长度为偶数：中心有 \(1\) 个
  • 直径长度为奇数：中心有 \(2\) 个

7.5 两棵树合并后的最小直径

3. 博弈论

3.1 SG 函数

解释

对于一个状态 x：

  • 先列出它一步能走到的所有状态，叫“直接后继”
  • 把这些后继状态的 sg 值收集起来
  • sg(x) 就是这些值的 mex

若游戏由若干个相互独立的子游戏组成，则总状态的 SG 值为各子游戏 SG 值的异或和：

\[ sg = sg_1 \oplus sg_2 \oplus \cdots \oplus sg_n \]

结论：

  • 异或和 != 0，先手必胜
  • 异或和 == 0，先手必败

3.3 拆分 Nim

若一个状态可以拆成两个独立子状态 i, j，则：

\[ sg(x \to (i,j)) = sg(i) \oplus sg(j) \]

4.2 Lucas 定理

适用场景： 模数 \(p\) 是质数，且 \(n,m\) 很大。

公式： \[ \binom{n}{k} \equiv \prod_i \binom{n_i}{k_i} \pmod p \] 其中 \(n_i,k_i\) 为 \(p\) 进制位。

4.3 插板法

核心模型： 将 \(n\) 个相同物品分给 \(k\) 个不同盒子。

  • 每盒至少一个：

\[ \binom{n-1}{k-1} \]

  • 允许为空：

\[ \binom{n+k-1}{k-1} \]

  • 每盒至少 \(m\) 个：

\[ \binom{n-km+k-1}{k-1} \]

  • 每盒至少 \(m_i\) 个，设 \(S=\sum m_i\)：

\[ \binom{n-S+k-1}{k-1} \]

4.4 多重集合排列

若总数为 \(n\)，第 \(i\) 种元素重复 \(n_i\) 次，则排列数为 \[ \frac{n!}{n_1!n_2!\cdots n_k!} \]

4.5 第二类斯特林数

含义： 将 \(n\) 个不同球放进 \(m\) 个不同盒子且每盒非空的方案数。

递推： \[ f[i][j]=j\cdot(f[i-1][j]+f[i-1][j-1]) \]

4.6 分拆数

定义： \(p(n,k)\) 表示把 \(n\) 拆成恰好 \(k\) 部分的方案数。

递推： \[ p(n,k)=p(n-1,k-1)+p(n-k,k) \]

4.7 范德蒙德卷积

\[ \sum_{i=0}^{k} \binom{n}{i}\binom{m}{k-i}=\binom{n+m}{k} \]

常见推论：

\[ \sum_{i=0}^{n} \binom{n}{i}^2=\binom{2n}{n} \]

\[ \sum_{i=1}^{n} \binom{n}{i}\binom{n}{i-1}=\binom{2n}{n-1} \]

\[ \sum_{i=0}^{m} \binom{n}{i}\binom{m}{i}=\binom{n+m}{m} \]

4.8 错位排列

前置知识补充：在 \(n\) 个元素的错排中，强制某个人传给特定另一个人的方案数，可以分类讨论为两人互传（剩下 \(n-2\) 错排）和两人不互传（相当于把这俩人绑成一个点，剩下 \(n-1\) 错排），总数为 \(D_{n-1} + D_{n-2}\)。

递推： \[ D_1=0,\ D_2=1,\ D_n=(n-1)(D_{n-1}+D_{n-2}) \]

4.9 卡特兰数

常见形式： \[ H_n = \frac{1}{n+1} \binom{2n}{n} \]

\[ H_n = \binom{2n}{n} - \binom{2n}{n-1} \]

\[ H_n = \frac{4n-2}{n+1} H_{n-1} \]

  • d < |r1-r2|：内含
  • d = 0 且 r1 = r2：重合

思路

设直线为 ab，圆为 O(c, r)。

做法：

  1. 先求圆心 c 在直线 ab 上的投影点 q
  2. 计算 q 到圆心的距离 d
  3. 比较 d 和 r
  4. 若有两个交点，则从 q 沿直线方向左右各走一段

返回值

  • 0：无交点
  • 1：一个交点
  • 2：两个交点

1.3 最大不相交区间数量

题型特征

给若干区间，要求选出尽可能多的区间，使它们两两不相交。

贪心策略

按右端点从小到大排序。

核心思路

这题和“区间选点”本质一样。
每次优先选右端点最小的区间，这样能给后面留下最多空间。

1.4 区间分组

题型特征

给若干区间，要求分成尽量少的组，使每组内部区间两两不相交。

贪心策略

按左端点从小到大排序，再用小根堆维护每组当前最小的右端点。

核心思路

  • 堆顶表示当前最早结束的那一组
  • 如果当前区间左端点 l <= heap.top()，说明和最早结束的那组也有冲突，只能新开一组
  • 否则可以放进这组，弹出旧右端点，加入新区间右端点

1.5 区间覆盖

题型特征

给定目标区间 [st, ed]，从若干区间中选尽量少的区间把它完全覆盖。

贪心策略

按左端点从小到大排序，每次在所有能接上当前 st 的区间中，选右端点最远的那个。

核心思路

设当前要覆盖到的位置是 st：

  • 找所有满足 l <= st 的区间
  • 在这些区间中选 r 最大的
  • 这样一步尽量向右扩展最多
  • 若最大 r 仍然小于 st，说明无法覆盖

模板

2. 反悔贪心

适用场景

有一批决策按顺序出现：

  • 当前先贪心选一个
  • 以后发现有更优选择时，允许撤销之前较差的选择
  • 典型维护结构通常是优先队列、平衡树

经典思路

用一个堆维护“当前已选方案中最差的那个”。 若新来的更优，就把最差的弹掉，再换成当前更优的。

做法理解

最小堆里维护“之前买入的股价”。 若当前价格 x 比堆顶更大，说明把之前更低价买的那股放到今天卖更优：

  • 收益增加 x - pq.top()
  • 弹出旧买入价
  • 插入当前价，表示这次卖出后仍可视为“在今天重新买回一股”，以便后续继续反悔
