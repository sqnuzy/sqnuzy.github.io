树的直径：

定义：树上任意两点间距离的最大值称为树的直径。通常用 \(D\) 表示长度，\((U, V)\)​ 表示端点。

方法一：两次 DFS
  • 优点：不仅能求出长度，还能求出具体的路径端点（\(U\) 和 \(V\)）。
  • 缺点：只适用于非负权边（如果边权有负数，这个贪心策略会失效）。
方法二：树形 DP 法

这种方法通过一次 DFS 遍历整棵树，在回溯的过程中计算。

  • 优点：
    1. 适用于 负权边。
    2. 只需要 一次 DFS。
    3. 容易嵌入到其他的树形 DP 逻辑中。
  • 缺点：
    • 如果不额外记录，很难直接知道直径的两个端点是谁，只能求出 长度。
1. 最远点性质（解题核心）

结论： 对于树上 任意节点 \(u\)，距离 \(u\) 最远 的节点 \(v\)，一定是树的某一条直径的 一个端点。

应用与推论：

  • 求直径（两次 BFS/DFS 法）：
    1. 任选一点 \(x\)，找到距离 \(x\) 最远的点 \(U\)（\(U\) 必为直径一端）。
    2. 从 \(U\) 出发，找到距离 \(U\) 最远的点 \(V\)（\(V\) 必为直径另一端）。
    3. \(U \to V\) 即为直径。
  • 求任意点的最远距离：
    • 只需要求出直径的两个端点 \(U, V\)。
    • 树上任意点 \(x\) 到树上其他点的最大距离 \(= \max(dist(x, U), dist(x, V))\)。
2. 树的中心 (Tree Center)

定义： 树上所有节点中，到树中其他节点 最大距离最小 的节点。

性质： - 树的中心一定位于 树的直径的中点。 - 一棵树最多有 1 个（直径为偶数）或 2 个（直径为奇数）中心。

求法算法：

  1. 先求出直径的两个端点 \(U, V\) 以及直径长度 \(len\)。
  2. 从 \(U\) 开始 BFS/DFS，记录每个节点的父节点 pre[]（用于还原路径）。
  3. 从 \(V\) 开始，通过 pre[] 往回跳 \(\lfloor len/2 \rfloor\) 步。
  4. 当前所在的节点即为树的中心。
3. 多条直径的交集性质

结论： 如果一棵树有多条直径，它们 一定相交，且所有直径的 交集部分 是一段连续的路径（或一个点）。

扩展应用（求所有直径端点）： - 如果题目要求统计有多少对点能构成直径，或者判断某条边是否在直径上。 - 思路： 1. 求出直径长度 \(L\)。 2. 从任意一个直径端点 \(U\) 出发遍历整棵树。 3. 所有满足 \(dist(U, x) = L\) 的点 \(x\)，都是直径的另一端点。 4. 反过来，从所有这些 \(x\) 出发，距离为 \(L\) 的点也是端点。

4. 树的合并与直径最小化

问题描述： 有两棵树 \(T_A\)（直径 \(D_A\)）和 \(T_B\)（直径 \(D_B\)），在两树之间加一条边，使得合并后的新树直径 最小。

核心思想： - 贪心策略：为了让新直径最小，应该把两棵树的 中心 连起来。 - 树的半径 \(R = \lceil D / 2 \rceil\)（即中心到最远点的距离）。

新直径公式： \[ D_{new} = \max \left( D_A, \quad D_B, \quad \lceil \frac{D_A}{2} \rceil + \lceil \frac{D_B}{2} \rceil + 1 \right) \] > 注：公式中的 \(+1\) 是因为连接两棵树的那条新边贡献了 1 的长度。

Boruvka算法

开始让每个点都成为一个连通块，接下来每一轮，对于每个联通快，维护块内块和块外点的最小边，然后连接这些边，合并连通块，直到只剩一个连通块。这样每一轮，最坏情况是连通块两两配对，连通块数量折半，实际还可能出现多个连通块在一轮里都连上，连通块减少的速度比折半还要快，所以至多运行 \(O(logn)\) 轮

题目描述

小苯拿到了一个由 \(n\) 个点、\(\tfrac{n \times (n-1)}{2}\) 条边组成的无向完全图，没有重边和自环，其中第 \(i\) 个点的点权为 \(a_i\)。\(u,v\) 两点之间的边权为\(a_u+a_v\)。

而调皮的小红删除掉了其中一些边，导致图不再是一个完全图，具体来讲有 \(m\) 条删除记录，第 \(j\) 条记录由一个点对 \((u_j,v_j)\) 组成，表示小红删除了这条边。注意，记录可能有重复，即可能存在两条记录删除的边是一样的。

现在你的任务就是求出这张图的最小生成树的权重（树中所有边的权重之和最小）。如果此时图不存在最小生成树，则输出 \(-1\)。

【名词解释】 最小生成树：对于一张由 \(n\) 个节点构成的连通图，选出 \(n-1\) 条边将所有节点连通，且使得这 \(n-1\) 条边的权重之和最小，这样的结构称为最小生成树。

每个测试文件均包含多组测试数据。第一行输入一个整数 \(T\left(1\leqq T\leqq 10^5 \right)\) 代表数据组数，每组测试数据描述如下：

第一行输入两个整数 \(n, m\left(2 \leqq n \leqq 3 \times 10^5;\, 1 \leqq m \leqq 3 \times 10^5 \right)\)，表示图的节点个数、小红删除边的记录条数。 第二行输入 \(n\) 个整数 \(a_1, a_2, \dots, a_n\left(1 \leqq a_i \leqq 10^9 \right)\)，表示每个节点的点权。 接下来 \(m\) 行，第 \(j\) 行输入两个正整数 \(u_j,v_j\left(1 \leqq u_j,v_j \leqq n;\, u_j \ne v_j \right)\)，描述第 \(j\) 条删除记录。

除此之外，保证单个测试文件的 \(n\) 之和、\(m\) 之和均不超过 \(3 \times 10^5\)。

输出格式

对于每一组测试数据，新起一行输出一个整数，如果此时图存在最小生成树，则输出整张图的最小生成树的权重；否则直接输出 \(-1\)。
